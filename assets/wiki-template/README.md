# __PROJECT__

Local knowledge base for the project. This page is the wiki home (the `README.md` in
`docs/` automatically becomes the index).

Knowledge lives in `.md` files under `docs/`, one file per theme. When a theme grows big
enough to stand on its own, create a dedicated page and add it to the `nav` section of
`mkdocs.yml`. Everything stays 100% local: no online publishing, no git unless it is
explicitly requested.

A few useful starting pages to create as needed:

- `architecture.md`: the structure, the modules, how the pieces fit together.
- `setup.md`: how to install and run, the prerequisites, the known pitfalls.
- `roadmap.md`: what's done, what's next, and the design decisions.
- `decisions.md`: the settled technical choices and their reasoning.
